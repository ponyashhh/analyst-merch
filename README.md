# Вебсайт для мерчу ІПСА "Analyst"

Це навчальний проєкт в рамках лабораторної роботи з дисципліни "Веборієнтована розробка програмного забезпечення". Сайт є вітриною для мерч-продукції Інституту прикладного системного аналізу (ІПСА).

## Структура проєкту

Проєкт організовано за стандартною структурою для веб-проєктів:

* `/css` — папка для файлів стилів (CSS).
* `/images` — папка для всіх зображень та графіки.
* `/js` — папка для JavaScript файлів (основна логіка у `main.js`).
* `index.html` — головна сторінка сайту.
* `discounts.html` — сторінка з акційними пропозиціями.
* `about.html` — сторінка з інформацією про проєкт та команду.
* `cart.html` — сторінка кошика.
* `t-shirts.html, shoppers.html, notebooks.html, pens.html, stickers.html` — сторінки категорій каталогу.
* `product-detail.html` — сторінка товару.
* `registration.html` — сторінка реєстрації та входу.
* `profile.html` — сторінка профілю користувача.

## JavaScript Функціональність

У рамках лабораторної роботи №2 до проєкту було додано скрипти для забезпечення інтерактивності та покращення користувацького досвіду (UX). Реалізовано наступні можливості:

1.  **Глобальні скрипти (працюють на всіх сторінках):**
    * **Темна тема:** Можливість перемикання між світлою та темною темами з запам'ятовуванням вибору користувача.
    * **Доступність:** Зміна розміру шрифту на сторінці за допомогою клавіш клавіатури (`ArrowUp` / `ArrowDown`).
    * **Навігація:** Підсвітка пунктів меню при наведенні та інтерактивне випадаюче меню.

2.  **Головна сторінка:**
    * **Динамічна дата:** Автоматичне відображення поточної дати у футері.
    * **Акордеон:** Функція "Читати далі" для розгортання/згортання великого обсягу тексту.
    * **Маніпуляція DOM:** Автоматичне виділення карток товарів (зелена рамка) та додавання динамічного текстового повідомлення в кінець сторінки.

3.  **Робота з формами (Profile Page):**
    * Клієнтська валідація даних перед відправкою.
    * Перехоплення події `submit` та виведення введених даних у консоль браузера.

## Використані технології та методи

### 1. DOM Manipulation (Робота з DOM-деревом)
Використовується для зміни вмісту та стилів сторінки в реальному часі.
* **Пошук елементів:** `document.getElementById()`, `document.querySelector()`, `document.querySelectorAll()`.
* **Зміна стилів:** Динамічна зміна CSS-властивостей (наприклад, `card.style.border` для карток товарів, `body.style.fontSize` для шрифтів).
* **Створення елементів:** Використання `document.createElement('p')` та `append()` для додавання нових блоків на сторінку без перезавантаження.
* **Класи:** Перемикання класів (`classList.toggle('dark-theme')`) для реалізації нічного режиму.

### 2. Events (Обробка подій)
Скрипти реагують на дії користувача:
* `DOMContentLoaded`: Гарантує, що скрипт запуститься тільки після повного завантаження HTML.
* `click`: Обробка кліків по кнопці зміни теми та кнопці "Читати далі".
* `submit`: Перехоплення відправки форми профілю.
* `keydown`: Відстеження натискання клавіш стрілок для керування масштабом тексту.
* `mouseenter` / `mouseleave`: Ефекти при наведенні миші на навігацію.

### 3. Form Validation (Валідація форм)
Реалізовано перевірку коректності введених даних за допомогою JavaScript:
* **Regular Expressions (Regex):** Перевірка формату телефону (`/^\d{9}$/`).
* **Логічні перевірки:** Перевірка довжини імені/прізвища, мінімальної довжини пароля (8 символів).
* **Порівняння:** Перевірка збігу полів "Новий пароль" та "Підтвердження пароля".
* **Error Handling:** Динамічне відображення повідомлень про помилки під відповідними полями (`showError`, `clearErrors`).

### 4. LocalStorage (Локальне сховище)
Використовується для збереження налаштувань користувача між сесіями:
* Збереження обраної теми оформлення (Dark/Light Mode).
* (Опціонально) Збереження імені користувача після успішного заповнення форми профілю.
